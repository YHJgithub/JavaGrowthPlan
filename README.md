# Java 成长路线图

面向 **1 年+ Java 后端经验** 的 9–10 个月进阶计划。以 **40 周、5 个阶段** 组织学习路径，涵盖 **100 项任务**（其中 **83 项必做**），附带推荐学习资源与可勾选进度追踪。

纯静态页面，零构建、零后端，打开即用；进度保存在浏览器本地，刷新不丢失。

## 在线体验

**[https://yhjgithub.github.io/JavaGrowthPlan/](https://yhjgithub.github.io/JavaGrowthPlan/)**

## 功能特性

### 学习规划

- **5 阶段路线图**：P1–P5 分段控制，每阶段独立进度条与完成率
- **按周划分章节**：手风琴式展开/收起，支持展开全部、收起全部
- **任务标签体系**：必做 / 选做 / 实践 / 并行，清晰区分优先级
- **推荐学习资源**：每章节附带网站、文章、书籍、视频等链接

### 进度追踪

- **整体进度**：顶部展示必做完成率与全部任务完成数
- **分阶段进度**：阶段卡片 + Tab 迷你进度条，一眼看到各阶段完成情况
- **章节进度**：每个章节显示必做项与全部项完成数
- **完成提醒**：章节、阶段、全部必做完成时弹出 Toast 提示
- **本地持久化**：勾选状态、当前阶段、章节展开状态自动保存至 `localStorage`

### 交互体验

- **iOS 风格界面**：分组列表、毛玻璃导航、圆角卡片与分段控件
- **粘性顶栏**：向下滚动后显示精简进度条，方便长页面浏览
- **阶段平滑切换**：基于 View Transitions API 的交叉淡入淡出（不支持时即时切换）
- **原生风格弹窗**：Alert 确认框与 Action Sheet 操作表（移动端「更多操作」）
- **主题切换**：深色 / 浅色 / 跟随系统，首屏无闪烁（`theme-init.js` 预加载）
- **移动端适配**：Safe Area 支持、触控反馈、振动反馈（支持的设备）
- **无障碍**：ARIA 属性、键盘 Esc 关闭弹窗、`prefers-reduced-motion` 降级

## 学习阶段

| 阶段 | 主题 | 周期 |
|------|------|------|
| Phase 1 | Java + Spring 核心夯实 | Month 1–2 · 8 周 |
| Phase 2 | 基础设施 + 数据库 + JVM | Month 2.5–4.5 · 9 周 |
| Phase 3 | 个人微服务项目 | Month 5–7 · 11 周 |
| Phase 4 | 深度技术提升 | Month 7.5–8.5 · 6 周 |
| Phase 5 | 求职冲刺 | Month 8.5–10 · 6 周 |

**技术栈侧重：** JDK 17 · Spring Boot 3 · Spring Cloud Alibaba

## 项目结构

```
JavaGrowthPlan/
├── index.html          # 页面入口（弹窗 / 操作表 DOM 骨架）
├── css/
│   └── styles.css      # 样式（主题变量、iOS 组件、响应式）

├── .gitignore

```

## 本地运行

无需安装依赖或构建，克隆后直接打开 `index.html` 即可。

若浏览器因 `file://` 协议限制脚本加载，可用任意静态 HTTP 服务：

```bash
npx serve .
```

或使用 VS Code / Cursor 的 Live Server 插件。

## 数据存储

| 键名 | 说明 |
|------|------|
| `java_growth_roadmap_v3` | 任务勾选、当前阶段、章节展开状态 |
| `java_growth_roadmap_theme` | 主题偏好（`dark` / `light` / `system`） |

数据仅存于本机浏览器，清除站点数据或更换设备/浏览器后进度不会同步。

## 修改路线图

编辑 `js/data.js` 中的 `PHASES` 数组即可增删阶段、章节与任务。任务字段说明：

| 字段 | 说明 |
|------|------|
| `id` | 唯一标识，用于进度存储 |
| `t` | 任务描述 |
| `optional` | 选做（不计入必做进度） |
| `practice` | 需动手实践 |
| `parallel` | 全程并行任务 |

## 部署

推送至 GitHub 后，在仓库 **Settings → Pages** 中选择 `main` 分支即可通过 GitHub Pages 发布（与在线体验地址相同）。
